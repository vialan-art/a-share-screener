# A-Share Screener

一个面向 A 股投资者的**研究型选股系统**。

在线地址：https://pizarro.vvnalnlgs.top

> 目标：用数据整理和量化呈现辅助投资学习，**不做买卖预测，不替代投资决策**。

---

## 一、项目定位与目标

本系统是一个"投研操作系统"的最小可行版本（MVP）。

对投资初学者来说，最大的痛点不是"没有数据"，而是：

- 数据太散，无法快速对比
- 财务指标概念复杂，难以建立直觉
- 不知道哪些公司是"明显不能碰"的
- 缺乏对研究过程的记录和复盘

因此，本系统的核心目标是：

1. **整理信息**：把股票基本面、估值、动量数据结构化
2. **量化呈现**：用打分和可视化建立对公司的直观认知
3. **排除风险**：通过可配置的及格线过滤掉有硬伤的公司
4. **辅助学习**：内置 AI 顾问，随时解释投资概念
5. **记录过程**：每日自动存档，支持历史对比

## 二、当前功能

### 数据层

- A 股股票列表、行情数据、财务指标采集
- 默认使用 **Mock 数据源**用于稳定演示，可一键切换为 **AkShare**（免费）真实数据
- 数据源接口可插拔，未来可替换为 Tushare、Wind、Bloomberg 等

### 过滤层

- 审计意见红线（只保留标准无保留意见）
- 行业差异化及格线（银行/保险/房地产等重资产行业放宽负债率要求）
- ROE、资产负债率、净利润增长、经营现金流等基础指标
- 过滤不通过会记录具体原因

### 评分层

- 三维度多因子打分：质量（45%）、估值（35%）、动量（20%）
- 使用百分位排名进行标准化，避免量纲差异
- 权重可在代码中调整，未来支持 UI 配置

### 可视化层

- 系统概览：快照日期、候选数量、最高分、更新状态
- 今日优选：TOP 股票列表与得分拆解
- 行业分布：候选股票的行业集中度
- 候选股票池：支持行业筛选、最低得分筛选、多维度排序
- 股票详情：雷达图 + 关键财务指标卡片
- 历史存档：每日快照对比
- 运行日志：数据更新状态追踪
- AI 顾问：OpenAI 兼容接口，可解释指标和选股逻辑

### 输出

- 导出 TradingView watchlist CSV
- 每日 19:00 自动运行并存档

## 三、选股逻辑：这个工具怎么帮你挑股票？

很多人说这只是一个"数据看板"——其实看板只是表皮，**真正的价值在于它把选股过程自动化、结构化、可复盘**。下面用大白话讲清楚它到底是怎么工作的。

### 1. 完整选股流程

```
采集数据 → 排雷过滤 → 多因子评分 → 生成今日候选池 → 存档复盘
```

- **采集数据**：每天拉取 A 股全市场财务指标（ROE、净利润、负债率、现金流等）和行情数据（PE、PB、涨跌幅、换手率、股息率）。
- **排雷过滤**：先剔除有明显硬伤的公司（审计非标、ROE 太低、负债率过高、现金流异常等）。
- **多因子评分**：对 surviving stocks 从质量、估值、动量三个维度打分。
- **生成候选池**：按总分排序，给出"今日优选"列表。
- **存档复盘**：每天保存一份快照，你可以回看某一天的选股结果，验证自己的判断。

### 2. 过滤规则（先排雷，再打分）

过滤就是"哪些公司直接排除"。规则分三类：

#### 通用红线（所有股票都要满足）

| 规则 | 默认值 | 为什么 |
|---|---|---|
| 审计意见 | 必须是"标准无保留意见" | 审计师都觉得报表没问题，这是最低门槛 |
| ROE | ≥ 5% | 投入资本长期赚不到 5%，投资价值很低 |
| 资产负债率 | ≤ 85% | 负债太高，抗风险能力差 |
| 有息负债率 | ≤ 75% | 借的钱太多，利息压力大 |
| 净利润增长 | ≥ -50% | 避免业绩断崖式下滑 |
| 经营现金流 | ≥ -10 亿 | 持续失血的公司先排除 |

#### 行业差异化规则

不同行业天然负债率不同，不能用同一根尺子量：

| 行业 | 特殊调整 |
|---|---|
| 银行 / 保险 | 负债率上限放宽到 95%，ROE 要求提高到 8% |
| 房地产 | 负债率上限放宽到 90% |
| 电力 | ROE 降低到 4%，负债率上限 80% |
| 煤炭 / 医药生物 | ROE 要求 8%，负债率收紧到 60%-70% |
| 食品饮料 | ROE 要求 10%，负债率收紧到 60% |

> 被过滤掉的股票会在"股票详情"里显示具体原因，比如"ROE 过低：2.3% < 5%"。

### 3. 评分机制（百分位排名 + 三维度加权）

过滤后剩下的公司进入打分环节。核心思想是：**不直接说"PE 15 好不好"，而是说"这只股票的 PE 在所有股票里排多少名"。**

这就是**百分位排名（Percentile Rank）**：把每个指标在所有股票中排序，最好的公司得 1.0，最差的得 0.0。

#### 三大维度

| 维度 | 权重 | 包含指标 | 指标方向 |
|---|---|---|---|
| **质量分** | 45% | ROE、毛利率、净利率、扣非净利润增长、经营现金流/净利润 | 越高越好 |
| **估值分** | 35% | PE-TTM、PB、股息率 | PE/PB 越低越好，股息率越高越好 |
| **动量分** | 20% | 涨跌幅、换手率 | 涨跌幅适中偏强、换手率不过低也不过高 |

#### 为什么权重是 45/35/20？

这是本系统的默认偏好，代表一种**"质量优先、估值合理、动量辅助"**的投资思路：

- **质量 45%**：好公司是第一位的。ROE 高、利润增长稳、现金流好的公司才进入候选。
- **估值 35%**：再好的公司买贵了也亏钱。PE、PB 低、股息率高说明相对便宜。
- **动量 20%**：市场短期情绪也会带来机会或风险。涨跌幅和换手率用来识别"近期有人关注但还没过热"的股票。

> 这些权重可以在 `backend/scoring/engine.py` 中调整，未来会支持前端直接拖动修改。

#### 评分示例

假设某只股票：
- ROE 18% → 在所有股票中排前 20% → 质量子项得 0.8
- PE 12 倍 → 在所有股票中偏低 → 估值子项得 0.75
- 今日涨 3%、换手率 4% → 动量子项得 0.6

系统会先算出质量分、估值分、动量分，再合成总分：

```
总分 = 0.45 × 质量分 + 0.35 × 估值分 + 0.20 × 动量分
```

### 4. 这个工具到底怎么帮你选股？（实际用法）

#### 用法一：快速缩小研究范围

A 股有 5000+ 只股票，你不可能每只都看。系统每天 19:00 自动跑一遍，把"排雷通过 + 综合评分高"的公司浓缩成一个几十只到两百只的候选池。你可以：

1. 打开 Dashboard 看"今日优选" TOP 20。
2. 按行业筛选，比如只看"食品饮料"或"医药生物"。
3. 点进某只股票，看它的财务指标、评分拆解、被过滤的原因。

#### 用法二：识别"便宜的好公司"

在列表里找这种组合：
- **质量分高 + 估值分高**：公司不错，还不算贵。
- **质量分高 + 估值分低**：可能被市场错杀，值得深入研究。
- **动量分突高**：近期有资金关注，但也要注意是否过热。

#### 用法三：避开常见陷阱

- **审计意见非标**：直接排除，不浪费感情。
- **ROE 低但股价涨很多**：动量分高但质量分差，可能是炒作。
- **负债率过高**：重资产行业有特例，但普通制造业负债 90% 要警惕。

#### 用法四：导出到 TradingView

点击"导出 Watchlist"，下载 CSV，导入 TradingView，可以直接在 K 线图上继续研究这些候选股。

#### 用法五：AI 顾问解释概念

看到"扣非净利润""经营现金流/净利润""有息负债率"不明白？直接问 AI 顾问，它会结合当前股票解释。

### 5. 数据源切换：Mock vs AkShare

本系统默认运行 **Mock 数据源**，数据是模拟生成的，只用于验证系统是否正常。要获得真实 A 股数据，切换到 **AkShare**：

#### 本地切换

```bash
cd backend
# 在 .env 中添加
SCREENER_PROVIDER=akshare
# 然后重新启动
PYTHONPATH=.. uvicorn backend.main:app --reload
```

#### Docker 部署切换

```bash
cd deploy
# 在 docker-compose.yml 或 .env 中设置
SCREENER_PROVIDER=akshare
docker-compose up -d --build
```

#### 手动触发一次真实数据选股

```bash
curl -X POST https://pizarro.vvnalnlgs.top/api/v1/run
```

> AkShare 数据来自东方财富等公开接口，**可能存在延迟、字段缺失或网络波动**。系统会在数据质量卡片中显示数据来源、完整度和更新时间，请把结果当作研究起点，不是买卖建议。

## 五、本地运行

### 1. 克隆项目

```bash
git clone https://github.com/vialan-art/a-share-screener.git
cd a-share-screener
```

### 2. 启动后端

```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 复制环境变量配置
cp .env.example .env

# 启动服务
PYTHONPATH=.. uvicorn backend.main:app --reload
```

### 3. 启动前端

```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:5173

### 4. 首次运行选股

在 Dashboard 点击"运行选股"，或调用 API：

```bash
curl -X POST http://localhost:8000/api/v1/run
```

## 十、已知问题

1. **AkShare 网络不稳定**：本项目默认使用 Mock 数据源保证演示稳定，生产环境可切换到 AkShare。
2. **行业分类不够精确**：AkShare 返回的行业字段在某些接口中为空，需要补充更可靠的行业数据源。
3. **评分权重固定**：目前需要在代码中调整，UI 配置尚未实现。
4. **缺少实时行情**：动量分基于当日行情，非高频数据。
